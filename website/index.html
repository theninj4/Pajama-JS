Intro Usage Clientside Serverside Inverted Browsers

TL;DR

Consider this example running either clientside (on an existing page), serverside (think Airbnb's Render project) or client-server hybrid (using PajamaJS to sync up the clients DOM with the serverside view's in realtime):


      var myModel = {
        counter: 0,
        increment: function() { this.counter++; }
      };
      var elem = new pjs.element(
        { tag: "div", innerHTML: { counter: myModel } },
        { tag: "input", type: "button", click: { increment: myModel } }
      );
      document.body.appendChild(elem.dom);

If that interests you (it should!) keep reading!

In a nutshell:

pajama.js (pjs) is a serverside and clientside templating engine which makes use of model-view data bindings to remove all HTML from your Javascript project and generally help you to achieve awesomeness.

This is what losing looks like:


      <input type="text" class="a b c" name="someInput" value="1234567" />

      <div class="float-right">
        <span>Total Price</span>
        <input type="text" name="priceInput" value="0.00" />
        <span class="small-text">(in pence)</span>
      </div>

and this is winning:


      { tag: "input", type: "text", classes: ["a", "b", "c"], name: "someInput" }

      { tag: 'div', classes: ['float-right'], contains: [
        { tag: 'span', text: 'Total Price:' },
        { tag: 'input', type: 'text', name: 'priceInput', value: '0.00' },
        { tag: 'span', classes: ['small-text'], text: '(in pence)' }
      ] }
